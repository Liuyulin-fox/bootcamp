一.pull request
１. とは:
分散バージョン管理システム(VCS)の機能の一つで、コードなどを追加・修正した際、本体への反映を他の開発者に依頼する機能である。
２. メリット:
・機能追加や改修など、作業内容をレビュー・マージ担当者やその他関係者に通知することができる。
・ソースコードの変更箇所をわかりやすく表示する。
・ソースコードに関するコミュニケーションの場を提供する。

ニ.GitHub flow
１. とは:
ブランチ戦略の1つであり、その名の通りGitHubで利用されているフローである
２. 構成:
メインのmasterブランチと機能開発のためのfeatureブランチ2つ部分である。
